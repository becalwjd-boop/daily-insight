# START_CHAT_PROMPT

아래 절차에 따라 Daily Insight News 프로젝트를 초기화하고 개발을 시작하세요.

━━━━━━━━━━━━━━━━━━━━━━
1. 프로젝트 초기화(Project Initialization)
━━━━━━━━━━━━━━━━━━━━━━

먼저 docs/README.md를 확인하여 프로젝트 문서 시스템을 이해하세요.

README.md는 프로젝트 문서 시스템의 Entry Point(Home)입니다.

README.md의 안내에 따라 필요한 문서를 순서대로 확인하여 프로젝트를 초기화하세요.

필요한 문서를 모두 읽기보다,

현재 작업(Task)에 필요한 문서를 우선적으로 확인합니다.

━━━━━━━━━━━━━━━━━━━━━━
2. 문서 확인(Document Initialization)
━━━━━━━━━━━━━━━━━━━━━━

README.md의 GPT Initialization Sequence를 기준으로 프로젝트를 이해합니다.

프로젝트 구조

현재 상태

운영 원칙

개발 규칙

최근 변경 사항

향후 작업

기술 부채

운영 정책

등 현재 작업에 필요한 문서를 스스로 판단하여 확인합니다.

━━━━━━━━━━━━━━━━━━━━━━
3. 개발 원칙(Development Principles)
━━━━━━━━━━━━━━━━━━━━━━

개발을 진행할 때는 아래 원칙을 따릅니다.

- 하나의 문서는 하나의 역할만 담당합니다.
- 프로젝트의 현재 상태는 MASTER.md를 기준으로 합니다.
- 변경 사항은 CHANGELOG.md를 기준으로 관리합니다.
- 앞으로의 작업은 NEXT_TASK.md를 기준으로 관리합니다.
- 기술 부채는 TECH_DEBT.md에서 관리합니다.
- 프로젝트 History는 REPORT에서 관리합니다.
- 모든 공식 문서는 DOCUMENT_RULES와 PROJECT_RULES를 따릅니다.

━━━━━━━━━━━━━━━━━━━━━━
4. 개발 진행(Development)
━━━━━━━━━━━━━━━━━━━━━━

프로젝트를 초기화한 후 개발을 진행합니다.

개발 중에는 필요한 공식 문서를 참고하며,

문서 간 역할이 중복되지 않도록 유지합니다.

새로운 기능을 추가하거나 기존 기능을 수정하는 경우,

관련 공식 문서의 업데이트가 필요한지도 함께 확인합니다.

━━━━━━━━━━━━━━━━━━━━━━
5. 세션 종료(Session Closing)
━━━━━━━━━━━━━━━━━━━━━━

개발이 종료되면

workflows/END_SESSION.md를 기준으로 프로젝트 종료 절차를 수행합니다.

필요한 공식 문서를 최신 상태로 업데이트하고,

REPORT 작성이 필요한지 확인합니다.

필요한 경우 docs.zip도 최신 상태로 갱신합니다.

━━━━━━━━━━━━━━━━━━━━━━
6. 프로젝트 운영 원칙(Project Operating System)
━━━━━━━━━━━━━━━━━━━━━━

이 프로젝트는 문서를 관리하는 프로젝트가 아니라,

Project Documentation System(Project Operating System)을 기반으로 운영됩니다.

새로운 개발자와 새로운 GPT는

별도의 설명 없이도 프로젝트를 이어갈 수 있도록 문서를 관리합니다.

문서보다 프로젝트 운영 원칙을 우선합니다.

README.md를 시작점(Home)으로 사용하여

프로젝트를 이해하고 개발을 진행하세요.

━━━━━━━━━━━━━━━━━━━━━━
7. 초기화 완료 보고(Initialization Report)
━━━━━━━━━━━━━━━━━━━━━━

프로젝트 초기화가 완료되면 아래 내용을 간단히 요약합니다.

- 확인한 문서
- 현재 프로젝트 상태
- 현재 우선순위
- 문서 운영 기준
- 다음 작업 진행 준비 여부

초기화 완료 후에는 아래 문장으로 마무리합니다.

"프로젝트 초기화가 완료되었습니다. 진행할 작업을 알려주세요."
